# Requirements Document

## Introduction

외부몰(네이버) 프로모션 성과 데이터를 자동으로 시각화하는 **네이버 프로모션 성과 대시보드** 시스템이다.
BXM 팀 내 외부몰 프로모션 담당자가 네이버 엑셀 파일을 업로드하면, 기획 의도 대비 실제 성과를 즉각적으로 비교 분석할 수 있는 인터랙티브 리포트를 생성한다.
담당자가 데이터 가공에 소요하는 시간을 줄이고, '의도 vs 결과' 분석 및 향후 액션 플랜 수립에 집중할 수 있는 환경을 제공한다. 채널은 네이버로 고정되며 별도 입력 불필요하다.

## Glossary

- **Dashboard**: 프로모션 성과 지표를 시각화하여 보여주는 메인 화면
- **Promotion**: 외부몰에서 진행하는 특정 기간의 판매 행사 (타임특가, 쿠폰 행사 등)
- **Channel**: 프로모션이 진행되는 외부몰 플랫폼 (현재 네이버 전용, 추후 확장 가능)
- **KPI_Card**: 핵심 성과 지표를 요약하여 보여주는 UI 컴포넌트
- **Excel_Parser**: 네이버 엑셀(.xlsx, .xls) 및 CSV(.csv) 파일을 읽어 표준 데이터 구조로 변환하는 모듈. 판매성과 파일, 상품성과 파일, 카테고리 파일 세 종류를 처리한다.
- **Sales_Performance_File**: 네이버에서 다운로드하는 판매 성과 파일. 일자별/시간별 매출 추이 분석에 사용한다. 주요 컬럼: 날짜, 요일, 시간대, 결제금액, 환불금액, 환불비율(결제금액), 쿠폰합계. 상품성과 파일과 독립적으로 사용하며 두 파일의 데이터를 합산하지 않는다.
- **Product_Performance_File**: 네이버에서 다운로드하는 상품별 판매 성과 파일. 주요 컬럼: 상품카테고리(대/중/소/세), 상품명, 상품ID, 결제수, 결제상품수량, 결제금액, 쿠폰합계, 상품쿠폰, 주문쿠폰, 환불건수, 환불금액, 환불비율(결제금액), 환불수량, 환불비율(결제상품수량)
- **Category_File**: 상품코드와 구분/대분류/상품명 매핑 정보를 담은 기준 카테고리 파일. 주요 컬럼: 상품코드, 구분, 대분류, 중분류, 소분류, 상품명(컬러제외). Category_File의 상품코드와 Product_Performance_File의 상품ID를 매칭하여 카테고리 분류에 사용한다.
- **Standard_Schema**: 네이버 파일 양식을 통일한 내부 데이터 구조
- **Planning_Intent**: 담당자가 행사 기획 시 설정한 목표 및 의도 (예: 신규 유입 증대, 재고 소진)
- **Performance_Result**: 행사 기간 동안 실제로 집계된 매출 및 행동 지표
- **AI_Analyzer**: 수치 데이터와 기획 의도를 결합하여 자동 코멘트 및 Next Action을 생성하는 모듈
- **Report**: 대시보드 분석 결과를 PDF/엑셀 형태로 내보낸 문서
- **Net_Sales**: 결제금액에서 환불금액을 차감한 최종 매출액
- **Achievement_Rate**: 목표 매출 대비 실제 Net_Sales의 비율 (%)
- **Refund_Rate**: 결제수 대비 환불건수의 비율 (%)
- **Time_Series**: 행사 기간 내 일별 또는 시간별로 집계된 매출 추이 데이터 (Sales_Performance_File 기반)
- **Peak_Time**: Time_Series 데이터에서 매출이 가장 높은 일자 또는 시간대

## Architecture Principles

- **데이터 저장 계층 분리**: 현재 구현은 브라우저 로컬 상태(메모리/localStorage)를 사용하되, 추후 백엔드 DB 연동이 가능하도록 데이터 저장 로직을 별도 레이어(Repository/Service 패턴)로 분리하여 구현한다.
- **DB 연동 준비**: 프로모션 레코드, 메모, 파싱된 데이터 등 영속성이 필요한 모든 데이터는 인터페이스를 통해 접근하며, 실제 저장소(localStorage ↔ DB)는 교체 가능한 구조로 설계한다.
- **디자인 시스템**: UI는 `DESKER_DESIGNGUIDE/` 내 디자인 가이드(색상 토큰, 타이포그래피, 컴포넌트 스타일 등)를 따른다. 상세 적용 기준은 design.md에 정의한다.
- **단일 페이지 플로우**: 모든 작업이 한 화면에서 이루어진다. 화면 상단에서 행사 정보를 입력하고 파일을 올리면, 그 아래로 스크롤하면 분석 결과(대시보드)가 바로 나타난다. 처음 접속한 사용자는 "① 행사 정보 입력 → ② 파일 업로드 → ③ 분석 결과 확인" 순서로 자연스럽게 진행할 수 있도록 각 단계에 번호와 안내 문구를 표시한다. 분석 후 입력값이나 파일을 수정하면 "다시 분석" 버튼이 나타나며, 버튼을 눌러야 대시보드가 갱신된다. 대시보드를 보는 중에는 상단 입력 영역을 접어서 화면을 넓게 쓸 수 있다.
- **AI API**: Requirement 7의 기획 의도 평가 및 Next Action 제언은 외부 AI API(예: OpenAI GPT)를 통해 생성한다. DB 연동 이후 함께 결정하여 구현한다.

## Future Scope (DB 연동 이후 구현)

다음 기능은 현재 구현 범위에 포함되지 않으며, DB 연동 완료 후 별도 단계에서 구현한다.

- **접근 암호 인증**: 사이트 접속 시 월별 암호를 입력해야 진입 가능한 인증 화면 제공
- **월별 암호 자동 갱신**: 매월 1일 자정에 암호를 자동으로 새로 생성하여 갱신
- **암호 자동 발송**: 갱신된 암호를 관리자가 지정한 이메일 주소로 자동 발송
- **행사별 메모**: 각 행사 분석 레코드에 연결된 개인 메모 기능 (행사 데이터가 DB에 저장된 이후 구현)
- **AI 자동 분석**: 기획 의도 텍스트와 수치 데이터를 결합하여 "의도가 실제로 구현됐는지" 평가하고 Next Action을 제언하는 기능. ChatGPT 등 외부 AI API 연동이 필요하며 API 키 비용이 발생하므로 DB 연동 이후 함께 결정하여 구현한다.
- **Confluence 전송**: 분석 완료된 대시보드 내용을 팀 Confluence 페이지에 자동으로 전송하는 기능. DB 연동 이후 백엔드 서버와 함께 구현한다. 상세 요구사항은 아래와 같다.
  - 분석 완료 후 대시보드에 "Confluence 전송" 버튼 표시
  - 버튼 클릭 시 백엔드 서버(Node.js 프록시)를 통해 Confluence REST API 호출
  - 지정된 Confluence 페이지(스페이스 키 + 부모 페이지 ID로 식별)에 행사별 섹션을 누적 추가
  - 전송 내용: 행사 컨텍스트(행사명, 기간, 기획 의도, 매출 목표) + KPI 수치 + 차트 이미지(html2canvas 캡처)
  - 전송 완료 후 DB에 전송 여부를 기록하고 버튼을 비활성화 (중복 전송 방지)
  - Confluence 연결 정보(Atlassian 사이트 URL, 스페이스 키, 부모 페이지 ID, API 토큰, 이메일)는 관리자가 서버 환경변수로 사전 설정하며 사용자는 별도 입력 불필요
  - WHEN Confluence 전송에 실패할 때, THE Dashboard SHALL "Confluence 전송에 실패했습니다. 잠시 후 다시 시도해주세요"라는 오류 메시지를 표시하고 버튼을 다시 활성화한다

## Requirements

### Requirement 1: 프로모션 컨텍스트 입력

**User Story:** 프로모션 담당자로서, 행사의 기본 정보와 기획 의도를 입력하고 싶다. 그래야 대시보드가 단순 수치가 아닌 의도 대비 결과를 보여줄 수 있기 때문이다.

#### Acceptance Criteria

1. THE Dashboard SHALL 행사명, 행사 기간(시작일/종료일), 라이브 진행일자(1회 이상 복수 입력 가능), 매출 목표 금액을 입력받는 폼을 제공한다. 채널은 네이버로 고정이므로 별도 입력 필드를 제공하지 않는다.
2. THE Dashboard SHALL 매출 목표 금액 입력 시 숫자를 3자리마다 쉼표로 구분하여 표시한다 (예: 100000 → 100,000).
2. THE Dashboard SHALL 기획 의도를 자유 텍스트로 입력받는 필드를 제공한다. 프로모션 유형 선택 필드는 제공하지 않는다.
3. WHEN 담당자가 필수 항목(행사명, 행사 기간)을 입력하지 않고 다음 단계로 진행하려 할 때, THE Dashboard SHALL 누락된 필드를 명시하는 유효성 오류 메시지를 표시한다.
4. WHEN 담당자가 종료일을 시작일보다 이전 날짜로 입력할 때, THE Dashboard SHALL "종료일은 시작일 이후여야 합니다"라는 오류 메시지를 표시한다.
5. THE Dashboard SHALL 라이브 진행일자를 날짜 추가/삭제 방식으로 2회 이상 입력할 수 있는 UI를 제공한다.
6. THE Dashboard SHALL 비교 행사 등록 시에도 이번 행사와 동일한 컨텍스트 항목(행사명, 행사 기간, 라이브 진행일자, 매출 목표 금액, 기획 의도)을 입력받는 폼을 제공한다.
7. WHEN 담당자가 컨텍스트 입력을 완료하고 저장할 때, THE Dashboard SHALL 입력된 컨텍스트 정보를 해당 프로모션 레코드에 저장하고 "✓ 저장됨" 피드백을 2초간 표시한다.
8. WHEN 담당자가 행사 정보를 저장하지 않은 상태에서 분석 시작 버튼을 클릭할 때, THE Dashboard SHALL "행사 정보가 저장되지 않았습니다. 저장 후 진행이 가능합니다. 저장하시겠습니까?" 확인 모달을 표시하고, 담당자가 확인을 선택한 경우 행사 정보를 저장한 후 분석을 진행한다. 담당자가 취소를 선택하거나 행사 정보 저장에 실패한 경우(필수 항목 미입력 등) 분석을 진행하지 않는다.
9. WHEN 분석이 완료될 때, THE Dashboard SHALL 분석 결과 섹션으로 자동 스크롤한다.

### Requirement 2: 파일 업로드 및 파싱

**User Story:** 프로모션 담당자로서, 이번 행사 파일을 필수로 업로드하고 필요시 최대 2개의 이전 행사 파일도 함께 업로드하고 싶다. 그래야 수작업 데이터 가공 없이 즉시 분석 및 비교를 시작할 수 있기 때문이다.

#### Acceptance Criteria

1. THE Dashboard SHALL 파일 업로드 화면을 "이번 행사(필수)"와 "비교 행사(선택, 최대 2개)" 섹션으로 구분하여 제공한다. 이번 행사 섹션은 판매성과 파일(필수), 상품성과 파일(필수), 카테고리 파일(선택)을 각각 업로드할 수 있는 영역으로 구성한다.
   - 판매성과 파일 다운로드 경로: 네이버 어드민 > 판매관리 > 주문통합검색 > 엑셀 다운
   - 상품성과 파일 다운로드 경로: 네이버 어드민 > 데이터분석 > 판매분석 > 상품성과 > 엑셀 다운
   - 카테고리 파일 다운로드 경로: 온라인 견적서 엑셀 시트
2. THE Dashboard SHALL 드래그 앤 드롭 및 파일 선택 방식으로 .xlsx, .xls, .csv 형식의 파일 업로드를 지원한다.
3. WHEN 담당자가 이번 행사 파일을 업로드하지 않고 분석을 시작하려 할 때, THE Dashboard SHALL "이번 행사 파일은 필수입니다"라는 오류 메시지를 표시하고 진행을 차단한다.
4. THE Dashboard SHALL 비교 행사 섹션에서 최대 2개의 행사 파일을 각각 독립적으로 업로드할 수 있는 UI를 제공한다.
5. WHEN 담당자가 파일을 업로드할 때, THE Excel_Parser SHALL 파일을 읽어 Standard_Schema로 변환한다.
6. THE Excel_Parser SHALL 네이버 파일 양식의 컬럼 매핑 규칙을 적용하여 Standard_Schema를 생성한다.
7. WHEN 업로드된 파일이 .xlsx, .xls, .csv 형식이 아닐 때, THE Dashboard SHALL "지원하지 않는 파일 형식입니다. xlsx, xls, csv 파일을 업로드해주세요"라는 오류 메시지를 표시한다.
8. WHEN Excel_Parser가 파일 내에서 필수 컬럼(상품ID, 결제금액, 주문일시)을 찾지 못할 때, THE Excel_Parser SHALL 누락된 컬럼명을 명시하는 파싱 오류를 반환한다.
9. WHEN 파일의 시계열 데이터 범위가 Requirement 1에서 입력한 행사 기간과 일치하지 않을 때, THE Dashboard SHALL 불일치 기간을 담당자에게 경고로 표시하고 계속 진행 여부를 확인한다.
10. THE Excel_Parser SHALL Standard_Schema로 변환된 데이터를 다시 원본과 동일한 구조로 재구성할 수 있어야 한다 (round-trip 속성).
11. WHEN Category_File이 함께 업로드될 때, THE Excel_Parser SHALL Category_File의 상품코드와 행사 성과 데이터의 상품ID를 매칭하여 구분/대분류/상품명 정보를 Standard_Schema에 결합한다.
12. WHEN Category_File이 업로드되지 않은 상태에서 상품별 성과 분석 섹션에 진입할 때, THE Dashboard SHALL "카테고리 파일이 없습니다. 카테고리 파일을 업로드하면 정확한 분류 기준으로 분석할 수 있습니다. 업로드하지 않으면 상품성과 파일의 상품카테고리(소)/상품명을 기준으로 표시됩니다"라는 안내와 함께 해당 화면에서 바로 업로드할 수 있는 버튼을 제공한다.
13. WHEN Category_File 없이 담당자가 계속 진행을 선택할 때, THE Excel_Parser SHALL 상품성과 파일의 상품카테고리(소) 및 상품명 컬럼을 대체 분류 기준으로 사용한다.

### Requirement 3: 핵심 지표(KPI) 카드 표시

**User Story:** 프로모션 담당자로서, 행사의 핵심 수치를 한눈에 파악하고 싶다. 그래야 빠르게 성과를 판단하고 보고할 수 있기 때문이다.

#### Acceptance Criteria

1. WHEN Standard_Schema 데이터가 로드될 때, THE Dashboard SHALL Net_Sales(결제금액 - 환불금액), Achievement_Rate, 최종 결제금액, 결제수, 쿠폰 사용금액(쿠폰합계), 환불액, Refund_Rate를 KPI_Card 형태로 표시한다. 쿠폰 데이터는 상품성과 파일의 쿠폰합계 컬럼을 사용한다. 모든 금액 수치는 3자리마다 쉼표로 구분하여 표시하며, KPI 카드 내 숫자가 잘리지 않도록 카드 너비를 충분히 확보하고 줄바꿈 없이 한 줄로 표시한다.
2. THE Dashboard SHALL Achievement_Rate를 "(Net_Sales / 매출 목표 금액) × 100"으로 계산하여 소수점 첫째 자리까지 표시한다.
3. THE Dashboard SHALL Refund_Rate를 "(환불건수 / 결제수) × 100"으로 계산하여 소수점 첫째 자리까지 표시한다.
4. WHEN Achievement_Rate가 100% 이상일 때, THE Dashboard SHALL 해당 KPI_Card를 목표 달성 상태임을 시각적으로 구분하여 표시한다.
5. WHEN Achievement_Rate가 100% 미만일 때, THE Dashboard SHALL 해당 KPI_Card를 목표 미달 상태임을 시각적으로 구분하여 표시한다.

### Requirement 4: 시계열 분석 차트

**User Story:** 프로모션 담당자로서, 행사 기간 내 매출 추이와 피크 타임을 그래프로 확인하고, 특정 일자를 클릭하면 해당 일자의 시간별 데이터도 볼 수 있기를 원한다. 그래야 어떤 시점에 프로모션 효과가 집중되었는지 정확히 파악할 수 있기 때문이다.

#### Acceptance Criteria

1. THE Dashboard SHALL 시계열 분석 차트 섹션을 Requirement 3(KPI 카드) 바로 아래에 배치한다.
2. WHEN Sales_Performance_File 데이터가 로드될 때, THE Dashboard SHALL 요일별(월~일) 집계 데이터를 최종결제액/결제금액/환불금액/쿠폰합계 순서로 테이블로 먼저 표시하고, 그 아래에 동일 데이터를 그래프로 표시한다.
3. WHEN 라이브 진행일자가 1개 이상 입력되어 있을 때, THE Dashboard SHALL 일별 추이 그래프에서 해당 일자에 라이브 방송 유무를 시각적으로 구분하여 표시한다.
4. THE Dashboard SHALL Time_Series 데이터에서 Peak_Time(일별 최종결제액 최고값 시점)을 자동으로 식별하여 그래프 위에 표시한다.
5. WHEN Sales_Performance_File에 시간대 컬럼이 포함될 때, THE Dashboard SHALL 행사 전체 기간의 시간대별 결제금액을 바 차트로 표시한다.
6. WHEN 담당자가 일별 그래프에서 특정 일자를 클릭할 때, THE Dashboard SHALL 해당 일자의 시간대별 결제금액을 바 차트로 전환하여 표시한다. 클릭한 날짜의 데이터만 필터링하여 표시한다.
7. THE Dashboard SHALL 시간대별 바 차트에서 전체 기간 보기로 돌아갈 수 있는 초기화 버튼을 제공한다.
8. THE Dashboard SHALL 상품별 성과 원형 차트에서 상위 4개 항목의 레이블과 비중(%)을 해당 파이 조각 위에 직접 겹쳐서 표시한다. 레이블은 파이 조각의 중심 방향에서 바깥쪽으로 선(labelLine)을 연결하여 어떤 조각에 해당하는지 즉시 인식할 수 있도록 한다.
9. THE Dashboard SHALL 평균 환불율을 초과하는 상품을 별도 "환불 참고 상품" 섹션으로 분리하여 표시한다. 섹션 제목 옆에 "평균 환불율을 초과한 상품으로, 다음 행사 기획 시 재고 및 품질 점검이 필요한 상품입니다"라는 설명을 표시한다. 해당 섹션도 구분/대분류/상품명 드롭다운 계층 구조로 표시한다.

### Requirement 5: 행사별 성과 히스토리 비교

**User Story:** 프로모션 담당자로서, 이번 행사와 이전 행사(최대 2개)의 성과를 동일 조건 기준으로 나란히 비교하고 싶다. 그래야 행사 효율의 개선 여부를 객관적으로 판단할 수 있기 때문이다.

#### Acceptance Criteria

1. WHEN 비교 행사 파일이 1개 이상 업로드될 때, THE Dashboard SHALL 이번 행사와 비교 행사(들)를 열(컬럼) 기준으로 나란히 배치하여 동일 지표(Net_Sales, Achievement_Rate, 주문건수, Refund_Rate 등)를 1:1로 비교할 수 있도록 표시한다.
2. THE Dashboard SHALL 최대 3개(이번 행사 + 비교 행사 2개)의 데이터를 동일 차트 및 테이블에 함께 표시할 수 있다.
3. WHEN 비교 행사 데이터에 이번 행사와 대응되는 지표가 없을 때, THE Dashboard SHALL 해당 셀을 빈칸으로 표시한다.
4. WHEN 비교 행사 파일이 업로드되지 않을 때, THE Dashboard SHALL 비교 섹션에 "이전 행사 파일을 업로드하면 비교 분석이 가능합니다" 안내 메시지와 함께 해당 화면에서 바로 파일을 업로드할 수 있는 업로드 버튼을 제공한다.
5. WHEN 담당자가 비교 행사 섹션을 직접 닫을 때, THE Dashboard SHALL 이번 행사 데이터만 전체 화면 너비에 맞게 확장하여 표시한다.

### Requirement 6: 상품별 성과 분석

**User Story:** 프로모션 담당자로서, 카테고리 파일 기준으로 상품별 판매 성과를 데이터와 그래프로 함께 확인하고 싶다. 그래야 다음 행사의 상품 구성 전략을 수립할 수 있기 때문이다.

#### Acceptance Criteria

1. WHEN Category_File이 업로드되어 있을 때, THE Dashboard SHALL Category_File의 구분 → 대분류 → 상품명 순서를 기준으로 상품 목록을 정렬하여 표시한다.
2. THE Dashboard SHALL Category_File의 상품코드와 행사 성과 데이터의 상품ID가 일치하는 항목의 합계를 집계하여 상품별 성과 데이터를 생성한다.
3. WHEN Standard_Schema에 상품별 데이터가 포함될 때, THE Dashboard SHALL 구분/대분류/상품명 기준으로 다음 지표를 테이블로 표시한다: 판매수량, 전체 대비 판매수량 비중(%), 상품 결제금액(결제금액 - 환불금액), 전체 결제금액 대비 결제액 비중(%), 환불건수, 환불률(환불수량 / 전체 주문수량 × 100).
4. THE Dashboard SHALL 구분값 기준으로 판매수량 비중과 결제금액 비중을 각각 원형(파이) 차트로 한 행에 크게 표시한다. 원형 차트는 잘리지 않도록 충분한 높이를 확보한다.
5. THE Dashboard SHALL 상품별 성과 테이블을 드롭다운 계층 구조로 표시한다: 기본 상태에서는 구분값만 표시하고, 구분값 클릭 시 해당 구분의 대분류 목록이 펼쳐지며, 대분류 클릭 시 해당 대분류의 상품명 목록이 정렬되어 표시된다.
7. THE Dashboard SHALL 상품별 성과 테이블을 판매수량, 상품 결제금액, 환불률 기준으로 정렬하는 기능을 제공한다. 정렬 기준 선택 시 구분/대분류/상품명 계층 전체에 걸쳐 해당 기준의 합계값 기준으로 내림차순 정렬한다.
8. WHEN Category_File의 상품코드와 일치하는 상품ID가 행사 성과 데이터에 없을 때, THE Dashboard SHALL 해당 상품을 "미매칭 상품"으로 분류하여 별도 표시한다.
9. WHEN 특정 상품의 환불률이 전체 상품 평균 환불률을 초과할 때, THE Dashboard SHALL 해당 상품을 "환불 참고 상품"으로 시각적으로 구분하여 표시한다.

### Requirement 7: AI 분석 — 기획 의도 vs 결과 및 Next Action *(Future Scope — DB 연동 및 AI API 결정 이후 구현)*

**User Story:** 프로모션 담당자로서, 내가 입력한 기획 의도가 실제로 구현되었는지 확인하고, 수치 변화가 큰 지표에 대한 자동 브리핑과 다음 행사에서 무엇을 더 살펴봐야 할지 추천을 받고 싶다. 그래야 데이터 해석 시간을 줄이고 의미 있는 회고와 다음 행사 준비에 집중할 수 있기 때문이다.

#### Acceptance Criteria

1. THE Dashboard SHALL 이 섹션을 대시보드 최하단에 배치한다.
2. THE Dashboard SHALL 담당자가 입력한 Planning_Intent 텍스트를 해당 섹션 상단에 표시한다.
3. WHEN 대시보드가 생성될 때, THE AI_Analyzer SHALL 담당자의 Planning_Intent와 이번 행사의 Performance_Result(매출, 주문건수, 쿠폰 사용, 환불률 등)를 결합하여 기획 의도가 실제로 구현되었는지 평가하는 코멘트를 생성한다.
4. THE AI_Analyzer SHALL 전체 KPI 지표 중 목표 대비 변화폭이 가장 큰 지표를 자동으로 식별하여 "이번 행사는 OO 지표가 특히 높았습니다"와 같은 브리핑 텍스트를 생성한다.
5. THE AI_Analyzer SHALL 이번 행사 데이터를 기반으로 다음 행사에서 유지할 전략과 개선할 전략을 각각 1개 이상 포함하는 Next Action 제언을 생성한다.
6. THE Dashboard SHALL 해당 섹션에서 유입, 재고, 전환율 등 이번 행사 데이터에 포함되지 않은 외부 지표는 분석 대상에서 제외한다.
7. WHEN AI_Analyzer가 외부 API 호출에 실패할 때, THE Dashboard SHALL 해당 섹션에 "AI 분석을 일시적으로 사용할 수 없습니다. 잠시 후 다시 시도해주세요"라는 메시지를 표시하고 나머지 대시보드 기능은 정상 동작한다.

### Requirement 8: 공유 코멘트 섹션

**User Story:** 프로모션 담당자 및 팀원으로서, 이 대시보드의 사용법이나 전체적인 안내 내용을 코멘트로 남기고 팀원들과 공유하고 싶다. 그래야 대시보드를 처음 접하는 팀원도 사용법을 쉽게 파악하고 소통할 수 있기 때문이다.

#### Acceptance Criteria

1. THE Dashboard SHALL 헤더 우측에 코멘트 아이콘 버튼을 상시 표시하여 사용자가 기능의 존재를 즉시 인식할 수 있도록 한다.
2. THE Dashboard SHALL 코멘트 아이콘에 현재 등록된 전체 코멘트 수를 뱃지로 표시한다.
3. WHEN 담당자가 코멘트 아이콘을 클릭할 때, THE Dashboard SHALL 화면 오른쪽에서 코멘트 사이드 패널이 슬라이드로 열린다.
4. THE Dashboard SHALL 사이드 패널 내에 이름 입력 필드와 코멘트 입력 필드를 제공하며, 이름을 입력한 누구나 코멘트를 작성할 수 있다.
5. WHEN 담당자가 이름과 코멘트를 입력하고 등록할 때, THE Dashboard SHALL 작성일시, 이름, 코멘트 내용을 함께 표시한다.
6. THE Dashboard SHALL 각 코멘트에 대한 리플(답글) 작성 기능을 제공하며, 리플에도 이름과 작성일시가 함께 표시된다.
7. THE Dashboard SHALL 코멘트 및 리플 데이터를 localStorage에 영속적으로 저장하여 브라우저를 닫았다가 다시 열어도 내용이 유지된다.
8. WHEN 이름 또는 코멘트 내용이 비어있는 상태에서 등록을 시도할 때, THE Dashboard SHALL "이름과 내용을 모두 입력해주세요"라는 오류 메시지를 표시한다.

### Requirement 9: 리포트 다운로드

**User Story:** 프로모션 담당자로서, 대시보드 화면 안에서 바로 분석 결과를 엑셀 또는 PDF로 내보내고 싶다. 그래야 별도 작업 없이 보고 및 팀 공유를 효율적으로 할 수 있기 때문이다.

#### Acceptance Criteria

1. THE Dashboard SHALL 대시보드 화면 내에 엑셀(.xlsx) 및 PDF 다운로드 버튼을 상시 표시한다.
2. 다운로드 출력 범위는 이번 행사 데이터만을 기준으로 하며, 비교 행사 데이터 및 공유 코멘트(Requirement 8)는 출력에 포함하지 않는다.
3. WHEN 담당자가 엑셀 다운로드를 요청할 때, THE Dashboard SHALL 다음 항목을 포함한 .xlsx 파일을 생성하여 다운로드한다: 행사 컨텍스트 정보(행사명, 채널명, 기간, 기획 의도, 매출 목표), KPI 수치, 일별/시간별 판매성과 데이터(날짜, 요일, 시간대, 결제금액, 환불금액, 최종결제액, 쿠폰합계), 상품별 성과 테이블. 엑셀 파일 내 모든 금액·수량 숫자는 3자리마다 쉼표로 구분하여 표시하고(예: 100,000), 비율 항목은 백분율 소수점 첫째 자리까지 표시한다(예: 36.4%).
4. WHEN 담당자가 PDF 다운로드를 요청할 때, THE Dashboard SHALL 다음 항목을 포함한 PDF 파일을 A4 기준으로 생성하여 다운로드한다: 행사 컨텍스트 정보(행사명, 채널명, 기간, 기획 의도, 매출 목표), KPI_Card, 시계열 차트, 상품별 성과 테이블.
5. THE Dashboard SHALL PDF 생성 시 각 섹션(KPI, 차트, 테이블 등)이 A4 페이지 경계에서 잘리지 않도록 페이지 나눔을 자동으로 처리한다.
6. WHEN PDF 다운로드 시 AI 분석이 실패한 상태일 때, THE Dashboard SHALL AI 코멘트 영역에 "AI 분석 결과를 불러올 수 없습니다"라는 텍스트를 포함하여 PDF를 생성한다.
7. WHEN 파일 생성 중 오류가 발생할 때, THE Dashboard SHALL "파일 생성에 실패했습니다. 다시 시도해주세요"라는 오류 메시지를 표시한다.
