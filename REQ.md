# 요구사항 (REQ)

> 상세 요구사항: `.kiro/specs/promo-performance-dashboard/requirements.md` 참조

## 프로젝트 개요

네이버 프로모션 성과 데이터를 시각화하는 대시보드.
BXM 팀 프로모션 담당자가 엑셀 파일을 업로드하면 기획 의도 대비 실제 성과를 분석하는 인터랙티브 리포트를 생성한다.

## 핵심 요구사항 요약

| ID | 요구사항 | 상태 |
|----|---------|------|
| REQ-1 | 프로모션 컨텍스트 입력 (행사명, 기간, 라이브일자, 매출목표, 기획의도) | 완료 |
| REQ-2 | 파일 업로드 및 파싱 (판매성과, 상품성과, 카테고리) | 완료 |
| REQ-3 | KPI 카드 표시 (순매출, 달성률, 결제금액, 결제수, 쿠폰, 환불액, 환불률) | 완료 |
| REQ-4 | 시계열 분석 차트 (일별/시간대별 추이, 라이브/피크 표시) | 완료 |
| REQ-5 | 행사별 성과 비교 (최대 3개 나란히) | 완료 |
| REQ-6 | 상품별 성과 분석 (구분/대분류/상품명 계층, 환불 참고 상품) | 완료 |
| REQ-7 | AI 분석 (기획 의도 vs 결과, Next Action) | DB 연동 후 → `REQ-DESIGN-TASK-DB.md` |
| REQ-8 | 공유 코멘트 섹션 | 완료 |
| REQ-9 | 리포트 다운로드 (엑셀/PDF) | 완료 |

## DB 연동 후 진행 항목

별도 문서로 분리: **`REQ-DESIGN-TASK-DB.md`** 참조
- REQ-7: AI 분석
- REQ-F1: 접근 암호 인증
- REQ-F2: 행사별 메모
- REQ-F3: AI 자동 분석
- REQ-F4: Confluence 전송
- REQ-F5: 행사명 DB 드롭다운
