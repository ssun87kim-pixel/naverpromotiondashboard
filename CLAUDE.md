# AI 개발 규칙

> 원본: `__coading rules/CODING-RULES.md`

## 필수 규칙

- 모든 응답은 한국어로 작성
- 이모지 사용 금지
- 파일 삭제/생성은 최소화, 기존 파일 편집 우선
- 요청하지 않은 리팩토링/기능 추가/개선 금지
- 변경하지 않은 코드에 주석/docstring/타입 추가 금지
- git commit/push는 명시적 요청 시에만 실행

## 확인 없이 하지 말 것

- 파일 삭제, git push/commit, 브랜치 삭제
- 패키지 설치/제거, 설정 파일 변경, 프로젝트 구조 변경

## 개발 프로세스

- 모든 개발은 REQ.md -> DESIGN.md -> TASK.md 순서로 문서 먼저 반영 후 구현
- 구현 완료 후 TASK.md 상태를 완료로 업데이트

## 푸터 규칙

- 모든 페이지 하단에 푸터 필수 표시
- 내용: 개발 및 수정문의: DESKER 김선영 | 버전명 | 배포일자(KST)
- 스타일: 가운데 정렬, 13px, #969696

## .gitignore 규칙

- .docx, .xlsx, .pdf, .png, .jpg, .jpeg, .gif, .bmp, .svg는 깃허브에 올리지 않음
- 예외: public/ 폴더의 앱 필수 이미지는 허용

## API 키 관리

- 민감 정보는 .env 파일에 저장, 코드에서 환경변수로 참조
- Vite: VITE_ 접두사 사용
