# 비교 행사 분석 구현 - 요구사항

## REQ-1: 비교 행사 파일 분석
- 비교 행사(최대 2개)도 단일 행사와 동일한 분석 로직 적용
- 판매성과 또는 상품성과 중 하나만 있어도 분석 가능

## REQ-2: 비교 행사 KPI 표시
- ComparisonSection에서 실제 분석 결과(compareKpis)를 표시
- 파일 없는 쪽의 KPI는 `-`로 표시 (영역은 유지)

## REQ-3: 비교 행사 닫기
- 비교 행사 닫기 버튼 클릭 시 해당 비교 데이터 제거

## REQ-4: 비교 행사도 메인 분석과 함께 실행
- "분석 시작" 버튼 클릭 시 이번 행사 + 비교 행사 파일 모두 분석
- ESC 취소도 동일하게 적용

## REQ-5: 부분 데이터 표시 규칙 (기존 메모리 반영)
- 단일 행사: 파일 없는 데이터의 영역은 제거
- 행사 비교: 영역 유지 + 없는 데이터는 `-` 표시
